Overview and Usage

This to-do planner is a implementation of the paper planner/to-do list but the user can simply keep track of their daily commitments on through this website. This to-do planner is built with "React" framework and currently achieve the basic functionality. The users can add new to-do items in the input box which will be queued at the end of the list, and the user can delete the task or toggle the checkbox to mark the task as done. Based on the amount of the remaining tasks, the website will remind users how many remaining tasks are left inside the to-do list.

To experience this virtual to-do planner, you may access it from To-Do-Planner Prototype

Issue Deep-Dive

One issue that took me a while to figure out is identifying when did the player hit the obstacle, with the .overlap() method, which means it should result into the "game over" screen. However, there was no such behavior when the player hits the obstacle and the two blocks will simply overlap with each other. Later on, under the help with my classmates, I have added another varible called "gameOver" to update the status of if the player loses the game and that will be the boolean check to identify if the console should end the game now. Additionally, my classmates have pointed out that in Javascript, boolean indicator DO NOT start with captalized letter, which means I should have written "gameOver = true" instead of "True". With these fixes, my code eventually start to behave almost correctly when the player hits the obstacle, and the rest of the problem was simply adjusting the position of the text and background colors.
