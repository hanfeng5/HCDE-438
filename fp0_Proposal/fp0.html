FP0 - Final Project Proposal

Description

This project is intented to be a schedule planner to help the user to plan out their calendar commitments and to-dos better for each day/week to reach a higher producitivty and organized schedule.

A sample schedule planner

Tools reuqired

  • Node.js
  • React
  • Potentially Electron

Versions of Prototype

For the project, I have broken the deliverable into two versions with the intial woring prototype and the refined final deliveable.

Prototype Version

In this verison, the user can simply add their to-do list for the day. The user can add contents to the daily to-do list and able to mark it as done to delete the to do item. The system will keep track of these data so that user can click the plus button at the bottom of the page to append one item to the list.

Final Version

This version consist of the single day schedule planner but spread out to multiple days of the week. The user can assign different to do for the next one week. Therefore, the system will be tracking of user's current day of the week and populate spaces for one week's to-do inputs.

Proposed Development Process

  1. Do the background research about React and understand what are the components/command needed to create the basic functionality.
  2. Start to build the basic to-do list. The user should be able to write in their own to-do content, add additional to-do items, or mark it as done.

    3. ----- MILESTONE #1 -----

  3. Then, add additional functionality that the user can put additional to-do items for other days of the week. It should apply the same concept of single day to do list but under different days of the week.
  4. Try to figure out how to save the changes to the schedule and save the progress for the to-list so that the user can check back later with certain authentication mechanism.

    5. ----- MILESTONE #2 -----

  5. Improve the user interface and imrpove the styles to make the page aesthetically pleasing!

    6. ----- FINAL DELIVERABLE -----

Known and Unknowns

Known: I have alreayd leanred the basic logistics for Javascript, HTML, and CSS. I have also interacted with the basic program development with npm. This includes the local development server, the build tool, and a deployment tool.

Unknowns: The biggest unknown is how to use the React library to create interfaces. I have never used React before which means that I need to learn about the syntax and the functionality of different command. Then, I need to figure out a way to layout the page into different blocks so that different parts of the page is running different javascript code potentially. This is different from the typical top to down page layout that information shows up in a large column, my deliverable has different functionality at various location on the page. I also need to figure out how to save the user data so that they can return to their progress later.

Stretch Goals

Essentailly, I want to add account authentications so that user can sign in with their Google account and potentially store the data into their account. Potentially, my schedule planner can automatically sync up the commiments from user's Google calendar and import it into the to-do list. This is a very big stretch goal and I may move some of the previous expected functionality into the stretch goal section if the progress turned out to be overly challenging than I have anticipated.
