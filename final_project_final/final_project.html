Final Project! To-Do Planner

Overview and Usage

This to-do planner is a implementation of the paper planner/to-do list but the user can simply keep track of their daily commitments on through this website. This to-do planner is built with "React" framework and currently achieve the basic functionality. The users can add new to-do items in the input box which will be queued at the end of the list, and the user can delete the task or toggle the checkbox to mark the task as done. Based on the amount of the remaining tasks, the website will remind users how many remaining tasks are left inside the to-do list. The user can also edit the existing task name if they desire.

To experience this virtual to-do planner, you may access it from To-Do-Planner

Issue Deep-Dive

One of the biggest issue during this stage is to figure out how to manipulate the task remaining number when the user checking and unchecking the checkbox without deleting the task. The old logic was to check how many task remains in the tasklist and the task remaining is simply the length of the task list. However, this method does not work anymore because the number should go down without deleting the task itself. After experimenting with all kinds of variable, I have thought about checking how many task are considered as "completed" becuase I have already developed the method to change the complete state to the opposite when the user interact with the checkbox. The hardest part is to pass this variable into a global scope and write out the condition based on the state changes to each individual task inside the tasklist. Finally, I have came up with the proper solution. Although I know it is not the moost efficient solution with the hard coded for loop, it does the job for a simple to-do planner.

Ideas and Future Work

Looking ahead, the first thing I wanted to do is to implement a sign-in page for user authentication. It was a stretch goal for the project and I have been trying to bring in Fireabse authentication into the app, there are a lot of nuances that I did not expect to work with React environemnt and the implementation is outside of the time constraint. Therefore, I want to allow user to sign into their Google or email account so that the information is stored with their account. In this case, the user will not lose their to-do data when they refresh the page or come back to the page at a later time. The most ambitious goal is to utilize these user authentication and automatically sync up the calendar invite into the to-do planner. In this case, the user can simply add and delete items directly connected to their clalendar.
