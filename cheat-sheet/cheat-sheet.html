CSS Styles:

  • justify-content: center -- This will align the contents, such as a group of pictures, to the middle of the page, or the container if specified.
  • nav a {transition: all 0.5s ease-out;} + nav a:hover, nav a:focus {background-color: #FA9500;color: #333;} -- First, this command will only apply to the anchor elements within the nav property. Then, it is saying when the user hover the mouse, or focus on the element, the CSS styles will be having "these" colors. But first, there need to be a "transition" declaration within the nav a styles to indicate the desired effect in changing between two states when the user trigger this CSS transition.
  • font-family: 'Fredoka One', cursive; -- This specifies the font that will be used within the corresponding container. The ones with "" is usually the special font that may require additional installation. It is usually followed by a "basic" font to tell the website to dispaly this alternative solution if the previous desired font is not available on user's side.
  • To make a grid-like layout: .tile-grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(20rem, 1fr)); /* grid-template-columns: repeat(auto-fit, 200px) 300px; */ gap: 1rem;
  • To make the picture adpative to the screen size .list_picture{ width: auto; max-width: 100%; height: auto; display: flex; aspect-ratio: 1; margin-bottom: 1rem; margin: 1rem; object-fit: contain; } the key is to set the width to auto and set the maximum width to 100 percent so that it will not over stretch the picture

Javascript

  • addEventListener: addYesNoButton.addEventListener("click", (e) => updateContentScript(true,false)); addOption.addEventListener("click", (e) => updateContentScript(false,true));
  • the .map function that will return a new data structure: function toggleTaskComplete(id) { const updatedTasks = tasks.map ((task) => { if (id === task.id) { return {...task, completed : !task.completed} } // console.log(remaining) return task; }) setTask(updatedTasks); }
  • the forEach method will perform the task on each individual instances: function countComplete () { remaining = 0; tasks.forEach(task => { if (!task.completed) { remaining += 1; } }) return remaining; }
  • A conditional statement: const task_noun = task_number !== 1 && task_number !==0? "tasks" : "task" This line is saying about assign the constant task_noun to different value based on the condition. If task number is not equal to 1 and task number is not equal to 0, go ahead and assign the value as "tasks". Otherwise, assign the value as "task".
