P5 Games "Avoid That!"

Overview and Usage

This project inovles using P5.js and P5.play to develop a simple interactive game that users can play from their web browsers. Players will be a small rectangular box at the bottom of the screen and the goal is to use the arrow key to move around the box to avoid the obstacle falling from the sky. This is a survival game that the player should try to last as long as possible in this game. This game is build with P5.js and P5.play libraries, and please feel free to learn more about the libraryies!

Development Process

  1. First, I have been doing reading through tutorials about how to make different sprites and create groups of sprites as the ground blocks for the game. Initially, the game was just a screen of sprite columns becuase I want to start by practicing how to make the simplest item in the game.

    A column of sprites as the first step

  2. Then, I started to play around how can I manupate the sprites location and the amount in order to create the obstacle feeling. By surprise, I have been playing around the createSprite() method and plugging into different numbers as experiments. Then, I have realized that if I do not follow the strict formula to connect sprites into a column, there will be a overloading effect that sprites are not normally organized into a linear form, and this looks like an "explosive effect". Therefore, I have decided to move this effect up on the top of the screen to potentially create a falling effect. One suggestion that I have received from my peer was that I should create three sprung point to create the obstacle instead of one, becuase this will fill the white spaces a bit better and give player some challenges to continue playing the game. With some calculation, I have identified three sprung points that are equally spread out from the player's browser window.

  3. Then, I have figured out how to add gravity and velocity to the sprites in order to create the falling effect. To make the game a bit harder for the users, I have made the sprites falling at random speed so that the users have to avoid the obstacle strategically. Additionally, I have added the feature that more obstacles will be added after 8 seconds if the game is still ongoing, and this is to increase the diffculty level for the player to stay alive.

    Explode at three spots
    Re-explode effect

  4. Finally, I have been spending most of the time to figure out how to use the keyboard to move the player sprite that I have created and how to end the game properly when the player sprite is overlapped with obstacles. My goal is to allow user to move "diagonally" if they hold down two direction key at the same time, such as up and left arrow will allow user to move diagonally towards the top left. I have been looking at different YouTube videos for the answer. Then, I have also been debugging about how to end the game promptly when the player "hit" one of the obstacle and provide an information that displays the highest score. My classmates have been helping me a lot in this debugging process. Finally, I have made a version that I am generally happy about.

Issue Deep-Dive

One issue that took me a while to figure out is identifying when did the player hit the obstacle, with the .overlap() method, which means it should result into the "game over" screen. However, there was no such behavior when the player hits the obstacle and the two blocks will simply overlap with each other. Later on, under the help with my classmates, I have added another varible called "gameOver" to update the status of if the player loses the game and that will be the boolean check to identify if the console should end the game now. Additionally, my classmates have pointed out that in Javascript, boolean indicator DO NOT start with captalized letter, which means I should have written "gameOver = true" instead of "True". With these fixes, my code eventually start to behave almost correctly when the player hits the obstacle, and the rest of the problem was simply adjusting the position of the text and background colors.
